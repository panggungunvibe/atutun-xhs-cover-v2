# atutun-xhs-cover-v2

阿囤囤风格小红书封面提示词生成工作流说明，推荐作为 Codex Skill 使用。博主本人尝试过 WorkBuddy、Trae、Kimi Work，目前都还没有成功跑起来。因此如果你希望稳定复现，建议先使用 Codex。

这个 Skill 用于把文章、脚本、选题、产品测评、AI 工具教程、效率方法、播客推荐、个人经验等内容，整理成可直接交给图片生成模型使用的小红书封面提示词。v2 版的核心变化是把收集流程收敛为 5 个必要问题：人物、辅助素材、标题、封面风格、标题用色偏好；其余视觉细节由模型根据内容自动具体化。


## v2 相比 v1 的优势

v2 的主要优势是大幅缩减问题数量。v1 更像完整的封面参数问卷，会让用户依次选择风格、人物动作、素材数量、素材用途、背景、字体、标题颜色和标题等细节；v2 只保留 5 个真正需要用户决定的问题：人物、辅助素材、标题、封面风格、标题用色偏好。

这样用户不用继续回答大量细碎视觉参数，agent 会根据内容主题、素材复杂度和封面风格自动具体化人物动作、背景、字体、配色、清单、贴纸、箭头和构图。结果是流程更短，但最终提示词仍然保持足够具体，可以直接用于图片生成模型。

## 适用场景

- AI 工具教程封面
- 产品测评和榜单封面
- 效率工作流封面

- 播客、资源、工具种草封面
- 个人经验和方法论封面

## 使用方式

### 在 Codex 中使用

把仓库中的 `atutun-xhs-cover` 目录复制到 Codex skills 目录：

```bash
cp -R atutun-xhs-cover ~/.codex/skills/
```

安装后可以在 Codex 中通过下面的方式调用：

```text
$atutun-xhs-cover
```

或直接描述需求，例如：

```text
用 atutun-xhs-cover 帮我生成一张 AI 工具测评小红书封面提示词。
```

### 在其他 agent 中使用

把 `atutun-xhs-cover/SKILL.md` 的内容作为系统提示词、项目指令、agent profile、workflow prompt 或知识库指令导入。核心要求是让 agent 遵守 `SKILL.md` 中的五问流程和最终提示词复核规则。

这条路径目前只是理论兼容说明，不保证所有 agent 都能正确执行。如果目标 agent 支持文件型工具/技能，可以保留目录结构；如果只支持纯文本 prompt，可以尝试直接复制 `SKILL.md` 正文。

## v2 使用流程

Skill 会按顺序提问，并且一次只问一个问题：

1. 上传人物参考图。
2. 上传辅助素材图，或回复“无”。
3. 从 3 个候选封面标题中选择，或要求重新生成。
4. 选择封面风格，默认可交给模型推荐。
5. 提供固定标题用色参考截图，或回复“无”。

收集完信息后，Skill 会根据用户选择的风格，或在用户选择“交给模型推荐”时自动挑选最合适模板，并输出完整、结构清晰、比例明确、可直接用于图片生成模型的封面提示词。

## 封面风格

目前包含 8 种封面风格：

1. 爆款大字压顶风
2. 巨字拆分冲击风
3. 小白科普问答风
4. 教程清单风
5. 产品测评榜单风
6. 种草推荐风
7. 贴纸拼贴入门风
8. 黑底效率工作流风

## 设计规范

- 默认生成 3:4 竖版小红书封面。
- 用户确认满意后，可扩展 4:3 和 16:9 横版，但必须重新适配构图。
- 真人出镜，人物通常占画面 35%-55%。
- 主标题必须超大、清晰、短促，并带粗黑描边或深色描边。
- 如果用户提供固定标题用色参考，优先贴近参考截图中的标题主色、关键词强调色和描边逻辑。
- 如果用户没有固定标题用色，标题色由模型根据画面自动选择；没有固定黄色参考时，凡使用黄色统一指定为柔和浅黄色 `#FDFFA7`。
- 人物边缘需要明显抠图描边。
- 画面强调高对比、高密度、强情绪和一眼可读。
- 重要文字、人物脸部和清单内容不能超出安全区。
- 输出前需要复核比例、人物、手部、文字、素材、配色和占位符。
